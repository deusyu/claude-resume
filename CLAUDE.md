# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

A resume management system: structured experience data → AI-tailored LaTeX → PDF.

## Skill

This project is distributed as a Claude Code Skill. The full skill definition lives in `skills/resume/SKILL.md`.

### Slash Commands

- `/resume init [text]` — Initialize experience data (guided Q&A or paste-to-import)
- `/resume generate [job title or JD file]` — Generate tailored resume → PDF
- `/resume add [description]` — Add/update experience data

### Build

```bash
make en      # English PDF (resume.pdf)
make zh      # Chinese PDF (resume-zh.pdf)
make all     # Both
make clean   # Remove build artifacts
```

Requires **XeLaTeX** (`brew install --cask mactex` on macOS).

## Architecture

```
experiences/  →  /resume generate  →  resume.tex  →  make en  →  output/Name-Title.pdf
   (素材)         (AI 定制化)          (LaTeX源)      (编译)        (输出)
```

### Directory Structure

- **`experiences/`** — Experience data (single source of truth)
  - `profile.md` — Personal info, contact, education
  - `work/` — Work history, one file per company
  - `projects.md` — Personal projects
  - `skills.md` — Skill inventory
  - `honors.md` — Honors + quantified metrics
- **`jobs/`** — Target JD files
- **`output/`** — Generated PDF output
- **`.history/`** — Auto-backup of previous .tex versions
- **`resume.tex`** — Current active LaTeX source
- **`resume.cls`** — LaTeX class definition
- **`skills/resume/`** — Skill definition + assets

### Key Rules

- **Keep resume to 1 page** (Chinese version), English version may extend slightly
- Backup `resume.tex` to `.history/` before each generation
- Keep raw details in `experiences/` — trimming happens at generation time
- `resume-zh.tex` is auto-generated by Makefile, do not edit manually
- Use frontmatter `tags` in experience files to help match job requirements
